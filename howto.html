This guide aims to be a step by step guide to getting started with Saucery for NUnit. If you find anything confusing or unclear, please contact us and we'll make the necessary updates.

Let's begin!

Installation

  1. Create a class library project in Visual Studio. It is recommended that this be part of your "System Under Test" solution.
  2. Right-click and select Manage NuGet Packages for Solution...
  3. Search for Saucery in the Online section and click install
  4. Add it to the class library project you created in step 1 above and click OK. Note: Only one project is to reference Saucery in your solution. Otherwise Saucery will not work correctly.

Activation

  1. Upon installation (and every time you open the solution until it is activated), the activation dialog will appear.
  2. If you have not already done so, purchase as activation key from the Full Circle website.
  3. Reenter the same details into the Activation dialog. These details should be entered in exactly the same.
  4. Click OK.
  5. If you entered the details correctly and validation was passed, a file called Saucery.lic will have been written to your solution directory. Commit this file to source control as Saucery will be looking for it at runtime.

Writing your tests

Your test classes should be placed in the Tests directory and have the same namespace as GlobalSetup. Doing so will mean GlobalSetup executes once for your all your Test suites.

Let's have a look at what your test classes need to look like.

using NUnit.Framework;
using OpenQA.Selenium;
using OpenQA.Selenium.Support.UI;
using Saucery.Tests;
using Saucery.ValueSources;
using System;
using System.Runtime.CompilerServices;

namespace MyApp.MyAppTests.Tests {
    public class MyTests : SauceryBase {
        [MethodImpl(MethodImplOptions.Synchronized)]
        [Test]
        public void MyFirstTest([ValueSource(typeof(PlatformTestData), "GetPlatforms")] PlatformTestData platform) {
            Setup(platform);
	  //
	  // Insert your test code here
	  //
        }
    }
}
